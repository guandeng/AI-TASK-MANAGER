# MCP 状态感知功能说明

## 功能概述

AI Task Manager 的 MCP 工具现在具有**状态感知**能力,当读取任务或子任务时,会自动检查其状态,并在状态不是"进行中"时给出友好的提示和建议。

## 为什么需要状态感知?

在实际使用中,当我们查看其他项目的任务或子任务时,了解其当前状态非常重要:

- ✅ **已完成 (done)** - 可以安全参考完整实现
- 🔄 **进行中 (in-progress)** - 部分功能已实现,可参考已完成部分
- ⏱️ **待处理 (pending)** - 尚未开始,建议查看规划思路
- 📌 **延期 (deferred)** - 暂时搁置,需注意延期原因
- 🚫 **阻塞 (blocked)** - 被其他问题阻塞,需了解阻塞原因

## 支持的工具

### 1. showTask - 显示任务详情

**新增参数**:
- `checkStatus` (可选,默认 true): 是否检查并提示任务状态

**示例**:
```javascript
{
  "name": "showTask",
  "arguments": {
    "id": "3",
    "projectRoot": "/projects/my-app",
    "checkStatus": true  // 默认启用
  }
}
```

### 2. getSubtask - 获取子任务

**新增参数**:
- `checkStatus` (可选,默认 true): 是否检查并提示任务状态

**示例**:
```javascript
{
  "name": "getSubtask",
  "arguments": {
    "sourceProject": "/projects/old-app",
    "taskId": "2.3",
    "checkStatus": true  // 默认启用
  }
}
```

## 状态提示类型

### 1. 任务级别提示

对于主任务,会显示:
- 当前状态 (pending/in-progress/done/deferred/blocked)
- 子任务进度 (如: 3/5 子任务已完成)
- 状态相关的建议操作

### 2. 子任务级别提示

对于子任务,会显示:
- 当前状态
- 父任务信息
- 父任务状态
- 是否可以安全参考

## 状态图标说明

| 图标 | 状态 | 说明 |
|------|------|------|
| ⏱️ | pending | 待处理,尚未开始 |
| 🔄 | in-progress | 进行中 |
| ✅ | done | 已完成 |
| 📌 | deferred | 延期 |
| 🚫 | blocked | 阻塞 |

## 使用场景

### 场景 1: 参考其他项目的实现

```
用户: 看看项目 A 的支付功能是怎么实现的

AI: [检查状态] ⏱️ 这个功能还在待处理状态
    建议: 可以查看需求描述和实现计划,但代码尚未实现

用户: 那 project-B 的呢?

AI: [检查状态] ✅ 这个功能已完成
    建议: 可以参考完整的实现代码和测试用例
```

### 场景 2: 决定是否开始任务

```
用户: 我接下来该做什么?

AI: 建议处理任务 5: 用户认证系统

用户: 好的,查看任务 5 的详情

AI: [检查状态] ⏱️ 任务 5 当前是 pending 状态
    是否要开始执行? 我可以帮你更新状态为 "in-progress"
```

### 场景 3: 了解项目进度

```
用户: 列出项目 A 的所有任务

AI: [对每个任务检查状态]
    ✅ 任务 1: 项目初始化 (done)
    ✅ 任务 2: 数据库设计 (done)
    🔄 任务 3: API 开发 (in-progress, 60% 完成)
    ⏱️ 任务 4: 前端开发 (pending)
    ⏱️ 任务 5: 测试 (pending)
```

## 禁用状态检查

如果你不需要状态提示,可以在调用时禁用:

```javascript
{
  "name": "showTask",
  "arguments": {
    "id": "3",
    "projectRoot": "/projects/my-app",
    "checkStatus": false  // 禁用状态检查
  }
}
```

## 最佳实践

### ✅ 推荐做法

1. **让状态检查保持启用** - 默认就是启用的,可以帮你做出更好的决策
2. **关注状态提示** - 根据状态决定如何参考任务实现
3. **及时更新状态** - 开始任务时更新为 "in-progress",完成时更新为 "done"

### 💡 智能建议

当任务不是 "in-progress" 状态时,AI 会提供智能建议:

**待处理 (pending) 任务**:
- 查看任务详情和规划
- 查看依赖关系
- 考虑是否开始执行

**已完成 (done) 任务**:
- 可以安全参考实现
- 查看测试策略
- 了解遇到的问题和解决方案

**进行中 (in-progress) 任务**:
- 查看已完成的部分
- 了解当前进展
- 可以协助完成剩余工作

## 实现细节

### 技术实现

1. 读取项目的 `tasks/tasks.json` 文件
2. 解析任务或子任务的状态字段
3. 根据状态生成友好的提示消息
4. 将提示消息添加到输出前面

### 性能考虑

- 状态检查是轻量级操作,只读取 JSON 文件
- 不会显著影响响应速度
- 可以通过参数禁用以提高性能(通常不需要)

## 总结

状态感知功能让 AI Task Manager 更加智能和人性化:

✅ **自动检查状态** - 无需手动确认
✅ **友好提示** - 用图标和颜色区分状态
✅ **智能建议** - 根据状态提供操作建议
✅ **可配置** - 可以禁用状态检查
✅ **跨项目** - 适用于所有项目的任务

这使得跨项目任务参考和管理变得更加安全和高效!

## 示例 2: 查看已完成的子任务

### 用户输入
```
看看 /Users/guandeng/projects/old-app 任务 2.3 的实现细节
```

### AI 响应
```
✅ **状态提示** (来源: old-app)
子任务 "文件上传进度显示" (父任务: 文件上传功能) 当前状态: **done**

✅ **已完成**: 可以安全参考此实现

---

📋 子任务来源: /Users/guandeng/projects/old-app
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

任务 2.3: 文件上传进度显示
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 基本信息
  ID: 2.3 (子任务)
  父任务: 2. 文件上传功能
  状态: done ✅
  优先级: medium

📝 描述
  实现文件上传时的实时进度条显示,支持大文件和批量上传

🔍 实现细节
  1. 使用 XMLHttpRequest 替代 fetch 以支持上传进度监听
     - 监听 xhr.upload.onprogress 事件
     - 计算 percentage = (loaded / total) * 100

  2. 添加进度条 UI 组件
     - 使用 NProgress 库显示顶部进度条
     - 支持多个文件同时上传时分别显示进度

  3. 处理上传失败和重试逻辑
     - 失败后自动重试 3 次
     - 指数退避: 1s, 2s, 4s

  4. 支持多文件并发上传
     - 使用 Promise.all 并发处理
     - 限制最大并发数为 3

✅ 测试策略
  - ✅ 测试大文件上传 (100MB+)
  - ✅ 测试网络中断恢复
  - ✅ 测试并发 10 个文件上传
  - ✅ 测试进度条准确性

💻 关键代码片段
```javascript
const upload = (file, onProgress) => {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();

    xhr.upload.onprogress = (e) => {
      if (e.lengthComputable) {
        const percent = (e.loaded / e.total) * 100;
        onProgress(percent);
      }
    };

    xhr.onload = () => resolve(xhr.response);
    xhr.onerror = () => reject(xhr.statusText);

    xhr.open('POST', '/api/upload', true);
    xhr.send(file);
  });
};
```

💡 **建议**: 可以在当前项目中复用这个实现模式
```

## 总结

状态感知功能让 AI Task Manager 更智能:

✅ **自动提醒** - 无需手动检查状态
✅ **智能建议** - 根据状态给出操作建议
✅ **决策辅助** - 帮助选择正确的行动
✅ **进度可视化** - 清晰展示项目进度
✅ **风险预警** - 提示被阻塞或延期的任务

这让跨项目任务管理变得更加高效和安全!
